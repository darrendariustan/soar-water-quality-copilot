### Proposed Scaffolding

```text
soar-copilot/
├── .env                  # Stores secure EXA_API_KEY and OPENAI_API_KEY (Ignored by Git)
├── .gitignore            # Core rules ensuring environmental secrets and local virtual envs never leak
├── agents.md             # The active master prompt governing system and agent behaviors
├── frontend/             # Subdirectory housing the entire containerized Streamlit application
│   ├── Dockerfile        # Container recipe utilizing a multi-stage build and uv for fast compilation
│   ├── app.py            # Main execution script for the single-page operator dashboard interface
│   ├── pyproject.toml    # Python dependency specification managed ultra-fast via uv (torch, wntr, exa-py)
│   ├── uv.lock           # Locked dependency tree generated by uv to guarantee build parity
│   ├── data/             # Static engineering assets loaded directly into memory via Streamlit caching
│   │   ├── network.inp   # Synthetic EPANET water distribution network configuration file
│   │   └── weights.pth   # Pre-trained PyTorch physics-informed neural network model weights
│   └── src/              # Modular Python files handling the underlying analytical and agent logic
│       ├── inference.py  # PyTorch PINN model architecture definition and forward pass logic
│       ├── simulation.py # WNTR simulation wrappers to load and parse hydraulic states
│       ├── exa_api.py    # Exa API client logic for localized context retrieval queries
│       └── copilot.py    # Conversational agent logic orchestrating LLM calls and state interpretation
├── scripts/              # Cross-platform orchestration utilities
│   ├── start-server.sh   # Bash script to compile and launch the local Docker container (Mac/Linux)
│   ├── start-server.bat  # Windows batch command script to execute local container initialization
│   ├── stop-server.sh    # Bash script to safely teardown local Docker infrastructure (Mac/Linux)
│   └── stop-server.bat   # Windows batch command script to safely spin down local infrastructure
└── docs/
    └── plan.md           # The primary engineering roadmap with phase-by-phase success criteria
```
